반응형 네비게이션 — 햄버거 메뉴 만들기

모바일에서는 햄버거(☰) 버튼과 슬라이드 메뉴 + 아코디언 서브메뉴, 데스크탑(768px~)에서는 가로 메뉴바 + hover 드롭다운. 하나의 HTML로 두 가지 메뉴를 만드는 과정을 단계별로 실습합니다.

STEP 0

반응형 웹 핵심 개념 3가지

이번 실습에 필요한 개념만 빠르게 정리합니다.

① 뷰포트(viewport)와 meta 태그

뷰포트는 브라우저가 콘텐츠를 그리는 기준 화면 크기입니다. 모바일 브라우저는 이 태그가 없으면 뷰포트를 약 980px로 잡고 페이지를 축소해서 보여주기 때문에, 미디어 쿼리가 의도대로 동작하지 않습니다. 반응형의 전제 조건이에요.

<meta name="viewport" content="width=device-width, initial-scale=1.0">
속성역할
width=device-width뷰포트 폭을 기기의 실제 화면 폭에 맞춤
initial-scale=1.0처음 로드될 때 확대/축소 배율을 100%로 (해상도와는 무관, 줌 배율만 담당)

② @media — 앳 룰(at-rule)

:으로 시작하면 가상 클래스, @로 시작하면 앳 룰입니다. 앳 룰은 CSS 엔진에게 "이 블록을 어떻게 처리할지" 지시하며, @media는 그중 화면 조건에 따라 내부 스타일 적용 여부를 결정하는 조건부 그룹 규칙이에요. 미디어 쿼리는 뷰포트 폭을 기준으로 조건을 판단합니다.

/* 뷰포트 폭이 768px 이상일 때만 { } 안의 스타일 적용 */
@media (min-width: 768px) {
  .burger { display: none; }
}

③ 모바일 퍼스트(Mobile First)

기본 CSS를 모바일 기준으로 작성하고, min-width 미디어 쿼리로 큰 화면일 때의 스타일을 추가하는 방식입니다. 이번 실습도 이 순서를 따릅니다: 햄버거 메뉴(기본) → 가로 메뉴바(768px~).

참고 768px은 태블릿 세로 폭에서 유래한 값이지만, 태블릿부터는 가로 메뉴가 충분히 들어가므로 "좁은 화면용 특수 처리(햄버거)가 필요 없어지는 경계"로 널리 쓰입니다. Bootstrap·Tailwind 모두 768px(md)을 기준으로 사용해요. 단, 브레이크포인트는 기기가 아니라 내 콘텐츠가 깨지는 지점 기준으로 잡는 것이 원칙입니다.

STEP 1

HTML 구조 잡기

버거 버튼과 메뉴를 nav 안에 나란히 배치합니다. 서브메뉴가 있는 항목에는 has-submenu 클래스를 붙여서 CSS/JS가 구분할 수 있게 해요.

<header id="site-header">
  <div class="logo">🎬 무비 컬렉션</div>
  <nav>
    <div class="burger">≡</div>          <!-- ☰ 또는 ≡ (수학기호) -->
    <ul class="nav-menu">                 <!-- open은 JS가 붙임. HTML에 미리 쓰지 않기! -->
      <li><a href="#">홈</a></li>
      <li class="has-submenu">
        <a href="#">장르</a>
        <ul class="submenu">
          <li><a href="#">액션</a></li>
          <li><a href="#">로맨스</a></li>
          <li><a href="#">SF</a></li>
        </ul>
      </li>
      <li><a href="#">랭킹</a></li>
      <li><a href="#" class="btn-signup">회원가입</a></li>
    </ul>
  </nav>
</header>
흔한 실수: 테스트하다가 <ul class="nav-menu open">처럼 open을 HTML에 남겨두면, 페이지를 열자마자 메뉴가 펼쳐진 상태로 시작합니다. open은 JS가 클릭 시 붙였다 떼는 클래스이므로 마크업에는 쓰지 않아요.
STEP 2

슬라이드 메뉴 — transform: translateX

메뉴를 화면 오른쪽 밖에 대기시켰다가, open 클래스가 붙으면 제자리로 미끄러져 들어오게 합니다.

왜 display:none → block이 아니라 translateX인가?

display는 none/block 같은 키워드 값이라 중간값이 없어서 트랜지션이 불가능해요. CSS 트랜지션은 숫자 ↔ 숫자 사이에서만 동작합니다. translateX(100%) → translateX(0%)는 숫자 변화라서 부드럽게 미끄러지는 애니메이션이 가능해요.

header{
  position: relative;   /* .nav-menu의 absolute 기준점 */
}

.burger{
  font-size: 2rem;
  cursor: pointer;
}

.nav-menu{
  position: absolute;         /* header 기준으로 배치 */
  background-color: var(--dark);
  top: 50px;                  /* 헤더 높이만큼 아래 */
  left: 0px;
  width: 100%;

  display: flex;
  flex-direction: column;     /* 모바일: 세로 나열 */

  transform: translateX(100%);   /* 자기 너비만큼 오른쪽 = 화면 밖 */
  transition: transform 0.5s ease;
}

.nav-menu.open{
  transform: translateX(0%);     /* 원래 자리로 슬라이드 인 */
}

.nav-menu > li{
  padding: 5px 40px;
}
.nav-menu > li:last-child{     /* 회원가입 버튼 스타일 */
  background-color: var(--accent);
  border-radius: 5px;
}

참고 translateX(50%)로 바꿔보면 "자기 너비의 50%만큼 이동한 위치"에 걸쳐 보입니다 — %가 부모가 아니라 자기 자신의 크기 기준이라는 걸 확인해보는 실험으로 좋아요.

메뉴가 translateX(100%)로 화면 밖에 있는 동안 body에 가로 스크롤이 생길 수 있어요. body{ overflow-x: hidden; }을 추가하면 방지됩니다.
STEP 3

서브메뉴 아코디언 — max-height + overflow:hidden

"장르"를 클릭하면 하위 메뉴가 아래로 펼쳐지면서 다른 항목을 밀어내는 아코디언을 만듭니다.

원리: overflow:hidden이 max-height 밖을 잘라준다

max-height는 그 자체로 내용을 숨기지 못해요. 기본값 overflow:visible에서는 콘텐츠가 박스 밖으로 넘쳐서 그냥 보입니다. overflow:hidden이 함께 있어야 max-height를 넘는 부분이 잘려서 안 보이게 돼요.

/* 초기: 높이 0 + 넘치는 콘텐츠 잘라서 화면에서 숨김 */
.submenu{
  max-height: 0px;
  overflow: hidden;
  transition: max-height 0.4s ease-in-out;
}

/* open 클래스가 붙으면 펼침 */
.has-submenu.open .submenu{
  max-height: 150px;    /* 실제 콘텐츠 높이에 가깝게! (아래 설명) */
}

.submenu li{
  padding: 5px 20px;
  border-bottom: 1px solid var(--light);
}
.submenu li:hover{
  background-color: var(--light);
  color: var(--dark);
}

왜 display가 아니라 max-height인가? — 그리고 max-height의 성질

속성트랜지션이유
display: none↔block❌ 불가키워드 값 — 중간값 계산 불가
height: 0↔auto❌ 불가auto는 숫자가 아니라 목표값 계산 불가
max-height: 0↔150px✅ 가능숫자 ↔ 숫자
주의 1 — 값을 너무 크게 잡으면 열림/닫힘이 비대칭: 실제 콘텐츠가 120px인데 max-height:500px로 잡으면, 열릴 땐 120px 지점에서 이미 다 보여서 빨리 열린 것처럼 느껴지고, 닫힐 땐 500→120px 구간(빈 구간) 동안 아무 변화가 없다가 뒤늦게 닫혀요. 실제 콘텐츠 높이에 가까운 값을 쓰거나, JS로 scrollHeight를 읽어 지정하세요.
주의 2 — open 상태에 overflow:visible을 넣지 마세요: 닫히는 트랜지션 도중 콘텐츠가 잘리지 않고 삐져나와 보이다가 뚝 사라집니다. overflow:hidden은 열림/닫힘 모두 유지해야 해요.
주의 3 — max-height:0은 패딩을 못 접습니다: 요소 자신의 padding은 콘텐츠가 아니라서 max-height로 줄일 수 없고 overflow로도 잘리지 않아요(box-sizing:border-box여도 패딩+테두리는 최소 크기로 남음). 서브메뉴 여백은 자식 li의 padding으로 주는 게 안전합니다.

화살표(▼) 붙이고 회전시키기

.has-submenu > a::after{
  content: "▼";
  margin-left: 10px;
  display: inline-block;   /* inline이면 rotate 기준점이 틀어져 2시 방향을 봄 */
  transform: rotate(0deg);   /* 닫힘 기준값 명시 — none↔180deg는 역방향이 튐 */
  transition: transform 0.4s ease-in-out;
}
.has-submenu.open > a::after{
  transform: rotate(180deg);
}

화살표는 li::after가 아니라 a::after에 붙입니다. li에 붙이면 서브메뉴 ul까지 포함한 블록의 맨 끝(서브메뉴 아래)에 나와서 엉뚱한 위치가 돼요.

타이밍 함수 ease-in은 느리게 시작해 빠르게 끝나서 닫힐 때 "한 템포 쉬는" 느낌이 나요. 아코디언에는 양쪽 다 부드러운 ease-in-out이 적합합니다.

STEP 4

자바스크립트 — open 클래스 토글

CSS는 "open이 있으면 이렇게 보여라"까지만 담당하고, open을 붙였다 떼는 건 JS의 몫입니다.

const burger = document.querySelector('.burger');
const menu = document.querySelector('.nav-menu');

// 1. 버거 클릭 → 메뉴 슬라이드 토글
burger.addEventListener('click', (e) => {
  e.stopPropagation();            // 3번(바깥 클릭 닫기)과 충돌 방지
  menu.classList.toggle('open');
});

// 2. "장르" 클릭 → 서브메뉴 아코디언 토글
document.querySelectorAll('.has-submenu > a').forEach(link => {
  link.addEventListener('click', (e) => {
    e.preventDefault();           // # 링크 이동 방지
    link.parentElement.classList.toggle('open');
  });
});

// 3. (선택) 메뉴 바깥 클릭 → 모두 닫기
document.addEventListener('click', (e) => {
  if(!menu.contains(e.target) && !burger.contains(e.target)){
    menu.classList.remove('open');
    document.querySelectorAll('.has-submenu.open')
      .forEach(li => li.classList.remove('open'));
  }
});

classList.toggle('open')은 클래스가 있으면 떼고 없으면 붙여요. CSS 트랜지션은 클래스가 바뀌는 순간 자동으로 발동하므로, JS는 애니메이션을 직접 다루지 않고 상태(클래스)만 관리하면 됩니다.

STEP 5

@media — 데스크탑 가로 메뉴로 전환

768px 이상에서는 버거를 숨기고, 세로 슬라이드 메뉴를 가로 메뉴바로 되돌리고, 서브메뉴는 hover 드롭다운으로 바꿉니다.

@media (min-width: 768px){
  .burger{
    display: none;               /* 버거 숨김 */
  }

  .nav-menu{
    position: static;            /* absolute 해제 → 헤더 흐름 안으로 복귀 */
    flex-direction: row;         /* 주축을 가로로 */
    transform: none;             /* translateX(100%) 해제 — 안 하면 메뉴가 안 보임! */
    width: auto;
    gap: 20px;
  }
  .nav-menu li{
    padding: 0px 10px;
  }
  .nav-menu > li{
    position: relative;          /* 드롭다운(absolute)의 기준점 */
  }
}
절대 하지 말 것: .has-submenu{ max-height: 0px; } — 서브메뉴가 아니라 "장르" 메뉴 항목(li) 자체를 높이 0으로 찌그러뜨립니다. 숨겨야 할 대상은 항상 .submenu예요.

데스크탑 드롭다운: max-height 대신 visibility + opacity

데스크탑 서브메뉴는 position:absolute로 메뉴바 아래에 떠야 하는데, 이때 max-height:0 방식은 패딩·테두리가 화면에 남는 한계가 있어요(위 STEP 3 주의 3). 절대 위치 드롭다운은 visibility + opacity 조합이 정석입니다.

@media (min-width: 768px){
  /* max-height는 "콘텐츠 영역의 높이"만 제한한다.
     콘텐츠는 0까지 줄어들 수 있지만 padding·border는 남기 때문에
     (border-box여도 마찬가지 — 콘텐츠가 음수가 될 수는 없으므로)
     패딩이 있는 드롭다운은 max-height 대신 visibility+opacity로 숨긴다 */
  .has-submenu .submenu{
    position: absolute;
    left: 0; top: 34px;
    background-color: var(--dark);
    width: 140px;
    border-radius: 5px;
    padding: 10px;                 /* 이제 패딩 자유롭게 사용 가능 */

    max-height: none;              /* 모바일 아코디언용 max-height 해제 (auto는 없는 값!) */
    overflow: visible;

    visibility: hidden;            /* 안 보이고 Tab 포커스도 안 감 (접근성 ↑) */
    opacity: 0;
    transform: translateY(-8px);   /* 살짝 위에서 내려오는 연출 */
    transition: opacity 0.3s ease, transform 0.3s ease, visibility 0.3s;
  }

  /* hover와 클릭(.open) 둘 다 지원 */
  .has-submenu:hover .submenu,
  .has-submenu.open .submenu{
    visibility: visible;
    opacity: 1;
    transform: translateY(0);
  }
  .has-submenu:hover > a::after{
    transform: rotate(180deg);
  }
}
상황적합한 숨김 방식이유
모바일 아코디언 (static 흐름)max-height + overflow:hidden펼쳐질 때 아래 항목을 실제로 밀어냄
데스크탑 드롭다운 (absolute)visibility + opacity패딩 한계 없음, 포커스 처리 올바름
max-height: auto는 존재하지 않는 값이라 선언 자체가 무시되고, 기본 규칙의 max-height:0이 그대로 살아남습니다. 그러면 배경색이 "안 먹는" 게 아니라 칠할 박스 높이가 0인 상태가 돼요. 해제는 반드시 max-height: none.
STEP 6

디버깅 가이드 — 증상별 원인 찾기

이 실습에서 실제로 겪게 되는 대표적인 증상과 원인입니다.

증상원인해결
트랜지션 없이 뚝 나타남/사라짐display:none↔block을 같이 바꾸는 중display 제거, max-height 또는 opacity만 변경
max-height:0인데 띠가 보임요소 자신의 padding (max-height로 못 접음)padding을 자식 li로 이동, 또는 visibility 방식
화살표가 닫힐 때 2시 방향을 봄::after에 display:inline-block 또는 기준값 rotate(0deg) 누락둘 다 명시
빨리 열리고 늦게 닫힘max-height가 실제 콘텐츠보다 너무 큼실제 높이에 가깝게 or JS scrollHeight
닫힐 때 콘텐츠가 삐져나옴open 상태에 overflow:visibleoverflow:hidden 유지
배경색이 안 먹음max-height:auto(무효값) → 높이 0 유지max-height:none
데스크탑에서 메뉴가 안 보임모바일용 translateX(100%) 해제 누락미디어쿼리에 transform:none
메뉴 항목이 찌그러짐.has-submenu(li)에 max-height:0li가 아니라 .submenu를 숨길 것
페이지 열자마자 메뉴가 펼쳐져 있음HTML에 open 클래스 하드코딩마크업에서 open 제거
가로 스크롤 생김translateX(100%)로 화면 밖 대기 중body{overflow-x:hidden}

TIP 스타일이 안 먹을 땐 F12 → 요소 선택 → Styles 패널: 속성에 취소선이 있으면 다른 규칙에 덮어써진 것, 취소선 없이 적용됐는데 안 보이면 박스 크기가 0이거나 가려진 것.
